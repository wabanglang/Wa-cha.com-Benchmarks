## FORGE Loop

```text

O — Organize the current chat/project corpus into source-bound semantic units.
R — Rank candidate systems/subsystems by RWAL, ABS, gain, cost, risk, redundancy, portability, determinism, and execution value.
```

## Required behavior when WaForge is invoked

1. Pull or reference the latest available WaForge runtime from the canonical repo, branch, and release path when GitHub access is available.
2. Run WaForge against the current chat, project, uploaded files, pasted corpus, and any provided candidate systems/subsystems.
3. Extract complete Wa!-relevant semantic units.
4. Evaluate systems/subsystems using RWAL + ABS + ATOMIC + CANONICAL gates.
5. Compare each candidate against the current Wa! baseline.
6. Decide whether to append, replace, add-in, merge, hold, or discard.
7. Generate an evolved derivative runtime only when it improves overall Wa! proficiency without degrading fidelity, truth, safety, determinism, portability, or execution usefulness.
8. Emit final systems/subsystems as structured artifacts:
   - evolved derivative prompt
   - extraction dataset
   - subsystem decision matrix
   - 1dfa1cb operation array
   - RWAL scorecard
   - manifest
   - checksums
   - binary/base64 transfer form when useful
9. Push or prepare the resulting systems/subsystems back to the canonical repo when GitHub write access is available.
10. If direct binary release upload is unavailable, push text-safe artifacts, base64 transfer files, checksums, and reconstruction instructions.
11. Never claim model-weight ingestion, fine-tuning, LoRA, adapter training, or persistent AI memory unless actually performed.
12. Fail closed on missing repo access, missing files, unverifiable sources, or unsafe claims.
