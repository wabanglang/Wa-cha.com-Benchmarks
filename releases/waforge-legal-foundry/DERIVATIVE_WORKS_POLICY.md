# Derivative Works Policy

## Purpose

This policy defines how forks, adaptations, wrappers, datasets, prompts, binaries, and downstream packages should preserve source lineage and project integrity.

## Required derivative notices

Derivative works should include:

- original project name
- original author attribution
- source repository or release reference
- commit or package version where known
- list of material changes
- derivative maintainer name
- release date

## Official status

A derivative is unofficial unless the Project Owner approves it in writing.

## Compatibility claims

Compatibility claims should be accurate, testable, and limited to the version tested.

## Release records

Each derivative release should keep a manifest, checksum list, and change summary.

## No ownership confusion

Derivative maintainers must not claim ownership of original WaForge materials or present the derivative as the original project.
