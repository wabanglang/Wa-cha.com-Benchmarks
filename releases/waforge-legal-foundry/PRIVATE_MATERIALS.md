# Private Materials

## Scope

Private materials include non-public source files, unreleased prompts, account exports, business terms, private communications, access credentials, internal strategy records, and personal data.

## Handling

Private materials should be stored separately from public releases and should not be published unless the Project Owner approves release.

## Public release screen

Before publication, check for credentials, private contact details, unpublished business terms, private account exports, non-public third-party materials, internal security notes, and personal data not intended for publication.

## Access

Access should be limited to people with a project purpose and a written obligation to preserve privacy.

## End of access

When access ends, private project copies should be returned, deleted, or archived according to the Project Owner's instruction.
