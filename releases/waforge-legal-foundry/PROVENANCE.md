# Provenance

Every WaForge release should be traceable from source to artifact.

## Release records

Each release should include release name, version, date, maintainer, source files, source hashes where practical, generated files, artifact hashes, branch, commit, source terms reference, review status, and RWAL score.

## Review

Before release, confirm that the artifact has a manifest, has checksums, preserves attribution, identifies source lineage, states known limitations, and avoids false claims.

## Correction

If an error is found after release, publish a corrected release note and preserve an audit trail.
