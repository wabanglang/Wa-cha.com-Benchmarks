# Data and Corpus Rights

## Scope

This document applies to project datasets, prompts, transcripts, semantic units, RAG chunks, extraction outputs, manifests, model-adjacent files, and corpus-derived packages.

## Ownership and source separation

The project should separate:

- original project-authored material
- user-provided material
- third-party material
- generated summaries
- derived semantic units
- release metadata

## Corpus use

Corpus materials should be used only where the project has permission, a lawful basis, or a project-controlled source.

## Sensitive material

Private keys, credentials, personal secrets, private account exports, and non-public third-party materials should not be published in public releases.

## Derived datasets

Derived datasets should preserve source references, transformation notes, and provenance records.

## RAG and local AI packages

RAG bundles, JSONL files, embeddings, and local import packages should include a manifest describing source lineage and intended use.

## Removal and correction

If a source is later found to be unsuitable for release, the project should correct the manifest, replace the release when practical, and preserve an internal audit note.
